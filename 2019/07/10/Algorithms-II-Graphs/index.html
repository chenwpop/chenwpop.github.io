Algorithms II, Graphs

Author: Chen Wahng Date: Jul 10, 2019  15:06:35 Category: Algorithms

Undirected Graphs

Definitions

  • Graph: group of vertices pairwise connected by edges
  • Path: sequence of vertices connected by edges
  • Cycle: path whose first and last vertices are the same
  • Degree: number of edges connected with the vertex
  • Connected Component: a maximal set of connected vertices
  • Bipartite: whether a graph can be divided into 2 groups that each edge connects 2 vertices from different group
  • Euler Tour: a cycle that uses each edge exactly once
  • Hamilton Tour: a cycle that uses each vertex exactly once

In this blog, assume that

  • numeric vertices
  • no parallel edges
  • no self loop in the graph
  • |V|: number of vertices, |E|: number of edges

Graph Representation

  • Set of Edges: maintain a list of edges.
  • Adjacency Matrix: maintain a two dimensional V-by-V boolean array.
  • Adjacency List:maintain vertex-indexed array of lists. The most common used one in real world.

Practices for Undirected Graphs

  • Is a graph bipartite?
    Starting from a random vertex, traverse the graph with BFS, and color adjacent layer with different group. If no conflicts occurs, then the graph is bipartite.
  • Does a cycle exist?
    Starting from a random vertex, do DFS and keep track of all vertices in current path. If an on-path vertex is visited twice, then a cycle exists.
  • To find an Euler tour?

A graph is Eulerian iff all vertices have even degree. First, check whether the graph is Eulerian, and sort all vertices by degree in descending order. Starting from the vertex with largest degree, then visit the vertex with largest degree among its adjacent vertices. Keep track of visited edges to avoid using them twice. Repeat these steps until all edges are visited.

  • To get the diameter?
    First, run BFS with a random vertex, keep the farthest vertex, then run another BFS with it.

Directed Graphs

Directions

  • Digraph: abbreviation of directed graphs
  • In Degree: number of edges that points to this vertex
  • Out Degree: number of edges that points out this vertex
  • Strongly-connected Component: directed bidirectional paths exist for all pairs of vertices
  • Directed Acyclic Graph: the literal meaning. It may not be a tree, since each tree node has only one parent.

Graph Representation

Similar to undirected graphs, adjacency list is the most practical way. Notice that, in adj[v], we only store the edges that go from vertex v.

Digraph Traversal

Reachability

Starting from vertex v, find all reachable vertices along a directed path. The idea and implementation is identical to the DFS algorithm of undirected graph.

Typical usage of reachability problem is to analyze the program control flow. Check whether the exit is reachable to avoid infinite loop, find and remove unreachable code blocks to eliminate dead code. Another useage is the Garbage Collector of Java, it mark all reachable resources, and free the unmarked memory.

Multiple-Source Shortest Path

Given a group of vertices g, find the shortest path of all other vertices to any vertices in g. The idea is to run BFS to this graph, but at beginning, enqueue all vertices in g.

Kosaraju-Sharir Algorithm

Kosaraju is a Russian professor. Once upon a time he forget to bring the lecture notes to class, so he developed such an easy-to-use algorithm to teach it.
The basic idea is,

  1. Get the topological order on the reverse graph of G. The phrase topological order is quite confusing for me at first. Topological order for any digraph indeed exists, while the topological sort is impossible for cyclic graph. To get the topological order, simply run the topological sort algorithm on this graph.
  2. Run DFS in G, visit the unmarked vertices in topological order.
    A simple but elegant algorithm.
